MACHINE LEARNING
To let an AI play your game, you effectively have to turn citadel_of_pershu.c into a library that follows
the "Gymnasium" (formerly OpenAI Gym) pattern. You need to expose four specific things:
1. The Reset Function: A way for the AI to instantly restart the game to START_ROOM with a fresh GameState.
2. The Step Function: Instead of the main_game_loop waiting for getchar(),
    you need a function where the AI sends a command (e.g., 0 for North, 1 for South) and the function returns:
    ◦ The Observation: The current state (either the raw numbers or the text description).
    ◦ The Reward: A floating-point number. You might give +1.0 for picking up a key,
        -0.1 for each turn taken (to encourage speed), and +100.0 for reaching the end.
    ◦ The "Done" Flag: A boolean telling the AI if it died or won.
3. The Action Space: A clear mapping of numbers to actions. The AI doesn't type "NORTH"; it sends the integer 0.
4. The Observation Space: A structured way for the AI to "see" the world. If you want it to learn quickly,
    you give it the raw GameState variables. If you want it to be "human-like," you only give it the text strings
    you print to the console.


    OpenAI Gymnasium

    Gymnasium (formerly OpenAI Gym) is the de facto standard for developing and comparing reinforcement
    learning algorithms. It provides a unified API for environments, making it easy to swap out different game worlds
    or simulations while keeping your agent code consistent.
    You can read more about Gymnasium and find its official documentation here:
    • Gymnasium Official Documentation: https://gymnasium.farama.org/
    • GitHub Repository: https://github.com/Farama-Foundation/Gymnasium
    The documentation is excellent and includes tutorials on how to create your own custom environments,
        which is exactly what you'll be doing to interface with your C game.

    Python Frameworks for Testing C Code with ML
    To test your C code with ML in Python, you'll typically need two main components:
    1. A way to interface Python with your C code: This allows your Python ML agent to call your C game's functions
        (perform_action, check_game_over, set_silent_mode, etc.) and access the GameState struct.
    2. A Reinforcement Learning framework: This provides the algorithms and tools to train your ML agent.
    Here are the recommended frameworks:
    1. Python-C Interface: ctypes
    ctypes is a foreign function library for Python. It provides C compatible data types and allows calling functions
    in dynamic link libraries (DLLs/shared libraries) from Python. This is the most common and often the most
    straightforward way to interface Python with C code for this kind of application, especially if you're compiling
    your C game into a shared library (.so on Linux/macOS, .dll on Windows).
    How it works:
    • You compile your citadel_of_pershu.c (and mersenne_twister.c) into a shared library, e.g., libcitadel.so.
    • In Python, you load this library using ctypes.CDLL().
    • You define the argument types (argtypes) and return types (restype) for your C functions in Python.
    • You can then call your C functions directly from Python, passing Python objects that ctypes automatically converts
        to C types. You can also define Python classes that mirror your C structs (like GameState)
        to easily pass and receive complex data.

    2. Reinforcement Learning Framework: Gymnasium + Stable Baselines3
    Once you have your ctypes wrapper, you'll integrate it into a custom Gymnasium environment.
    • Gymnasium: As discussed, this provides the reset() and step() methods that your ML agent will interact with. Your custom Gymnasium environment will:
    ◦ __init__(): Load your C library and initialize GameState.
    ◦ reset(): Call your C initialize() function, reset GameState, and return the initial observation.
    ◦ step(action): Map the agent's integer action to your C perform_action() call (e.g., action=0 maps to perform_action(gs, 'N', 0,0,0)). It then calls perform_action(), check_game_over(), calculates the reward, and returns the next observation, reward, done flag, and info dictionary.
    • Stable Baselines3 (SB3): This is a set of reliable implementations of reinforcement learning algorithms in PyTorch. It's built to work seamlessly with Gymnasium environments. Once you have your custom Gymnasium environment, you can plug in various SB3 algorithms (like PPO, A2C, DQN) to train your agent.

    Why this combination?
    • Standardization: Gymnasium provides a consistent API for your environment, making it easy to experiment with different RL algorithms.
    • Powerful Algorithms: Stable Baselines3 offers state-of-the-art RL algorithms that are well-tested and optimized.
    • Community Support: Both are widely used, meaning plenty of documentation, tutorials, and community help.

    Conceptual flow:
    1. C Code: Your citadel_of_pershu.c compiled as libcitadel.so.
    2. Python Wrapper (ctypes): A Python module that loads libcitadel.so and exposes your C functions and GameState struct to Python.
    3. Custom Gymnasium Environment: A Python class that inherits from gymnasium.Env, using your ctypes wrapper to interact with the C game.
    4. RL Agent (Stable Baselines3): An SB3 algorithm (e.g., PPO) that learns to play your game by interacting with your custom Gymnasium environment.

    This setup provides a robust and efficient way to train an ML agent on your C-based text adventure.

lib = ctypes.CDLL("./libcitadel.so")

# Define the C GameState struct in Python

    _fields_ = [
        ("player_name", ctypes.c_char_p),
        ("room", ctypes.c_int),
        ("turns", ctypes.c_int),
        ("cash", ctypes.c_int),

        ("is_dead", ctypes.c_bool),
        ("completed", ctypes.c_bool),
        # ... MTState, items, rooms_visited ...
    ]

# Define argument and return types for C functions
lib.initialize.argtypes = [ctypes.POINTER(GameState)]

    ctypes.POINTER(GameState),
    ctypes.c_char,
    ctypes.c_int,
    ctypes.c_int,
    ctypes.c_int
]
lib.perform_action.restype = ctypes.c_bool

lib.check_game_over.argtypes = [ctypes.POINTER(GameState)]

lib.set_silent_mode.restype = None
